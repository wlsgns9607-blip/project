# 조진훈 포트폴리오 랜딩페이지 (App Developer & Designer)

이 저장소는 UI/UX 기획 및 애플리케이션 개발자 **조진훈**의 포트폴리오 랜딩페이지 소스 코드입니다. 
단순한 화면 디자인을 넘어, 실제 고객의 숨겨진 불편을 발견하고 AI 기술을 활용해 구조적인 해결 방향을 설계하는 과정을 담았습니다.

## 🌟 조진훈 기획자만의 3가지 차별점

### 01. 책상이 아닌 '야생의 불편함'과 '공간 분석'에서 발굴하는 기획력
뻔한 오픈 데이터나 통계 자료만 짜깁기하는 기획이 아닙니다. 직접 운전대를 잡으며 마주한 페인 포인트부터, 음식점 한 곳을 가더라도 공간의 인테리어, 분위기, 서비스, 주방 상태까지 본능적으로 관찰하고 분석하는 날카로운 시선을 가졌습니다. 이처럼 일상의 생생한 결핍과 오프라인 공간의 디테일 속에서 진짜 시장의 빈틈을 동물적으로 포착해 냅니다.

### 02. 단편적 아이디어를 '비즈니스 생태계'로 결합하는 스케일
단순히 "이런 기능이 있으면 좋겠다"에서 멈추지 않습니다. 메인 서비스의 흐름을 설계하는 동시에, 타깃을 확장하고 유관 상권이나 AI 기술을 유기적으로 결합하여 최종적으로 '회사의 지속 가능한 매출과 이익(BM)'을 만들어내는 구조를 짭니다.

### 03. 툴 숙련도를 넘어 '프로덕트의 본질'을 보는 시야
화려한 디자인 컴포넌트나 규격에 갇히지 않습니다. 서비스의 본질은 결국 '사용자 한 명의 행동 변화'를 이끌어내어 회사의 가치를 증명하는 것에 있음을 알기에, 최소한의 핵심 기능(MVP)으로 시장을 빠르게 장악하는 전략적 UX를 지향합니다.

### 💡 Overview : 협업에 대한 새로운 정의
진정한 협업은 팀원의 머릿수에서 나오는 것이 아니라, 도구를 완벽히 제어하여 결과물을 만들어내는 실행력에서 시작됩니다. 

저는 소통 비용이 큰 기존 방식 대신, LLM(Gemini, Claude)을 완벽한 파트너로 제어하며 기획부터 프론트엔드 배포까지 완비하는 **'AI 바이브 코딩(Vibe Coding)'** 프로세스를 수립했습니다.

---

### 🛠️ AI 협업 워크플로우 (My AI Work Process)

* **Step 1. 기획 & 가설 검증 [ Gemini 1.5 Pro ]**
  - 비즈니스 모델 타당성 분석 및 구조화
  - 타겟 페르소나 설계 및 여정 지도 도출
  - 예외 케이스(Edge Case) 및 서비스 정책 발굴
* **Step 2. UI/UX 디자인 디렉팅 [ Claude 3.5 ]**
  - 도메인 맞춤형 UI 구조 도출 (예: 시니어 특화, Quiet Luxury)
  - 컴포넌트 구조화 및 반응형 레이아웃 설계
* **Step 3. 코드 구현 & 즉시 배포 [ Google AI Studio ]**
  - 시맨틱 마크업(HTML5) 및 CSS Variables 기반 스타일 추출
  - 로컬 스토리지 데이터 연동을 통한 가설 검증용 MVP 구현 및 배포

### 🤝 AI 협업 경험이 '실제 사람과의 협업'으로 연결되는 방식 (Human Collaboration)
❓ **"AI와만 일해본 기획자는 현업에서 디자이너, 개발자와 어떻게 소통할 것인가?"**

이에 대해 저는 **AI 협업을 통해 단련된 '명확한 기획 고정력'**으로 실무 협업의 비효율을 제로(0)로 만들 것을 확신합니다.

1. **감정이 배제된 논리적 지시서(PRD) 작성 역량**
   - AI에게 원하는 결과물을 한 번에 얻어내기 위해 프롬프트를 쪼개고 다듬었던 과정은, 실제 개발자에게 '오해의 여지가 없는 엄격한 기능 명세서'를 제공하는 훈련이 되었습니다.
2. **상대방의 언어(도메인)로 대화하는 소통력**
   - 코드 구현을 위해 HTML/CSS 및 시스템 구조를 직접 핸들링하며 개발 지식을 습득했습니다. 현업에서 개발자에게 "이거 그냥 해주세요"가 아닌, "로컬 스토리지 데이터 바인딩 시 이 방식으로 분기 처리가 가능한가요?"와 같이 개발 친화적인 언어로 소통하여 개발 리드타임을 획기적으로 줄이겠습니다.
3. **소통 비용의 최소화와 빠른 의사결정**
   - 1인 PO로서 기획-디자인-배포 전체 사이클을 혼자 겪어봤기에, 실무 협업 시 각 파트(디자인, 개발)의 작업 공수를 정확히 예측하고 불필요한 회의와 커뮤니케이션 미스를 최소화할 수 있습니다.
